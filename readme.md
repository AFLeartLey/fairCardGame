# Project FairCard

Simple multiplayer Card Game based on python-socketio.

# 运行指南

安装依赖
```
pip install -r requirements.txt
```

运行游戏主窗口
```
python ./client/main.py
```

# 游玩指南

Project Faircard 的核心乐趣在于：每次你要抽的牌不是随机从牌库摸到，而是由对手在若干张随机生成的牌中“替你挑一张”，让你承担代价、获得奖励，并用资源管理把对手血量打到 0。

## 游戏准备

打开主界面后，你和你的对手应该在ip和端口的框内填入指向同一个ip的相同端口的内容以进行联机，并保证你们处于同一局域网内。接下来主机选择"创建房间"，客户端选择"加入房间"，完成后主机即可开始游戏。

## 游戏目标与胜利
- 当任意一方在打出某张牌后生命值归零，游戏立刻结束；将对手生命值降到 0 的玩家获胜。
- 对局由若干回合组成，双方轮流进行自己的回合。

## 牌的结构：代价与奖励
每张牌都由一组随机成对出现的“代价”和“奖励”,以及牌的等级构成；打出牌时先检查是否满足代价条件，满足后才能结算效果。等级越高，造成效果的数值量就越大。

**代价**（可能出现的类型）  
- 丢弃若干张手牌。
- 对自己造成若干伤害。
- 花费若干 Cost。

**奖励**（可能出现的类型）  
- 获得若干张手牌。
- 回复若干血量。
- 回复若干 Cost。
- 对对方玩家造成若干伤害。

## 回合流程（你与对手）
在整个对局中，界面会持续显示：日志栏、回合结束按钮、双方 Cost/血量/手牌信息（对手仅显示手牌数量；你方显示每张手牌详情，文本呈现为“（奖励），但是（代价）”）。

你的回合  
- 回合开始：你会获得一张牌，但这张牌是由对手从若干张随机生成的牌里选择后“发给你”的。
- 出牌阶段：你可以打出手牌；若将要出的牌不满足条件（例如需要弃 3 张牌但手牌不足、或 Cost 不够），该牌将无法打出。
- 结束回合：点击回合结束按钮后进入“做牌画面”，你需要从若干随机生成的牌中选一张发给对手，随后对手进入其回合，做牌画面消失。

对手回合  
- 你应能看到对手的日志同步更新。
- 当对手需要你来“替他选牌”（做牌需求）时，会弹出做牌画面；你完成选择后，画面消失并继续对局。

## 新手上手提示
- 抽牌由对手决定这一机制意味着：你需要准备“应对坏牌”的冗余资源（例如保留足够手牌数量来承受弃牌代价，或留出 Cost 缓冲）。
- 作为选牌者时，通常优先把“高代价、低收益”或“会卡住对手当前资源结构”的牌给对手（例如对手手牌少时给高弃牌代价，对手 Cost 紧张时给高 Cost 代价）。
